# 黑群晖系统安装

## 什么是黑群晖

黑群晖是指在非群晖官方硬件上安装群晖 DSM（DiskStation Manager）系统的方法。相比白群晖（官方硬件），黑群晖具有更高的性价比和硬件选择自由度，可以在普通 x86 硬件上体验到群晖系统的强大功能。

!!! warning "重要声明"
    黑群晖仅供学习和个人使用，不建议用于商业环境。请支持正版软件，如有商业需求请购买官方白群晖设备。

## 准备工作

### 系统要求

- **宿主机**：已安装 ESXi 的服务器
- **内存**：推荐至少分配 2GB 给群晖虚拟机
- **存储**：至少 20GB 用于系统安装
- **网络**：确保虚拟机能够正常联网

### 所需文件

1. **引导镜像**：ARPL 引导文件（推荐使用最新版本）
2. **DSM 系统**：群晖官方 DSM 系统文件（会自动下载）

## 创建虚拟机

### 步骤1：新建虚拟机

在 ESXi 管理界面中，点击 **创建/注册虚拟机**：

1. 选择 **创建新的虚拟机**
2. 输入虚拟机名称：`Synology-DSM`
3. 选择兼容性：**ESXi 7.0 及更高版本**
4. 客户机操作系统系列：**Linux**
5. 客户机操作系统版本：**其他 3.x Linux（64 位）**

### 步骤2：配置虚拟硬件

| 硬件组件 | 推荐配置 | 说明 |
|----------|----------|------|
| **CPU** | 2核 | 最少1核，推荐2核以上 |
| **内存** | 4GB | 最少2GB，推荐4GB以上 |
| **硬盘1** | 50GB | 系统盘，薄置备 |
| **硬盘2** | 500GB+ | 数据盘，根据需求调整 |
| **网络适配器** | VMXNET3 | 推荐使用 VMXNET3 |

### 步骤3：高级配置

在虚拟机选项中进行以下设置：

1. **引导选项**：
   - 固件：**BIOS**（不要选择 UEFI）
   - 引导延迟：**5000 毫秒**

2. **引导顺序**：
   - 设置从 CD/DVD 驱动器启动
   - 将引导镜像挂载到虚拟光驱

## 安装引导系统

### 步骤1：启动虚拟机

1. 挂载 ARPL 引导镜像到虚拟光驱
2. 启动虚拟机
3. 等待系统进入 ARPL 配置界面

### 步骤2：配置 ARPL 引导

在 ARPL 主界面中进行以下操作：

```bash
# 1. 选择 "Choose a model"
# 推荐选择：DS918+ 或 DS920+

# 2. 选择 "Choose a Build Number" 
# 选择对应的版本号（建议选择稳定版本）

# 3. 选择 "Choose a serial number"
# 可以生成随机序列号或手动输入

# 4. 添加必要的驱动和插件
# 在 "Addons" 中添加所需驱动
```

### 步骤3：构建引导文件

1. 选择 **"Build the loader"**
2. 等待引导文件构建完成
3. 构建完成后，选择 **"Boot the loader"**

## 安装 DSM 系统

### 步骤1：进入 DSM 安装

1. 虚拟机重启后，会自动从构建的引导启动
2. 等待系统加载，观察控制台输出的 IP 地址
3. 在浏览器中访问显示的 IP 地址

### 步骤2：DSM 安装向导

在浏览器中完成以下步骤：

1. **搜索 DSM**：
   - 系统会自动搜索局域网内的群晖设备
   - 找到后点击 **连接**

2. **安装 DSM**：
   - 选择 **手动安装**
   - 上传 DSM 系统文件（.pat 文件）
   - 或选择 **在线安装**让系统自动下载

3. **等待安装**：
   - 系统会自动安装并重启
   - 安装过程约需 10-15 分钟

### 步骤3：初始配置

DSM 安装完成后进行初始配置：

1. **创建管理员账户**：
   - 用户名：admin
   - 设置密码（建议使用强密码）

2. **网络设置**：
   - 配置静态 IP（推荐）
   - 设置 DNS 服务器

3. **存储配置**：
   - 创建存储池
   - 设置 RAID 模式（根据需求选择）
   - 创建存储空间

## 虚拟机优化配置

### 步骤1：移除安装介质

安装完成后，执行以下操作：

1. 关闭群晖虚拟机
2. 编辑虚拟机设置
3. 移除 CD/DVD 驱动器或设置为客户机设备
4. 确保引导顺序为硬盘优先

### 步骤2：网络优化

为了更好的网络性能，建议：

1. 使用 **VMXNET3** 网络适配器
2. 启用 **SR-IOV**（如果硬件支持）
3. 配置专用的 VLAN（可选）

### 步骤3：存储优化

1. **磁盘控制器**：
   - 推荐使用 LSI Logic SAS 控制器
   - 启用磁盘热添加功能

2. **存储分配**：
   - 系统盘使用厚置备（性能更好）
   - 数据盘可使用薄置备（节省空间）

## 存储配置建议

### RAID 配置方案

根据数据重要程度和性能需求选择：

| RAID类型 | 最少磁盘数 | 容错能力 | 空间利用率 | 适用场景 |
|----------|------------|----------|------------|----------|
| **Basic** | 1 | 无 | 100% | 临时存储 |
| **JBOD** | 2+ | 无 | 100% | 最大容量 |
| **RAID 1** | 2 | 1块磁盘 | 50% | 重要数据 |
| **SHR-1** | 2+ | 1块磁盘 | 灵活 | 推荐方案 |
| **SHR-2** | 4+ | 2块磁盘 | 灵活 | 高可靠性 |

### 存储池规划

```
推荐配置：
├── 存储池 1（系统+应用）
│   ├── 磁盘：2×1TB SSD（RAID 1）
│   └── 用途：系统、套件、缓存
├── 存储池 2（重要数据）
│   ├── 磁盘：4×4TB HDD（SHR-1）
│   └── 用途：文档、照片、重要备份
└── 存储池 3（媒体文件）
    ├── 磁盘：6×8TB HDD（SHR-1）
    └── 用途：影音文件、冷备份
```

## 常见问题解决

### 问题1：找不到群晖设备

**现象**：浏览器中无法找到群晖设备

**解决方案**：
1. 检查虚拟机网络连接是否正常
2. 确认虚拟机与客户端在同一网段
3. 尝试直接访问虚拟机IP地址:5000端口
4. 检查防火墙设置

### 问题2：DSM 安装失败

**现象**：DSM 系统安装过程中出错

**解决方案**：
1. 确认引导文件版本与 DSM 版本匹配
2. 检查虚拟机硬件配置是否满足要求
3. 重新构建引导文件
4. 尝试使用不同的 DSM 版本

### 问题3：启动后无法进入系统

**现象**：虚拟机启动后卡在引导界面

**解决方案**：
1. 检查引导顺序设置
2. 确认引导文件构建成功
3. 增加虚拟机内存分配
4. 检查存储控制器配置
